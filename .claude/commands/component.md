# /component

Create a reusable Blade UI component with Tailwind CSS styling.

## Instructions

The user will describe a UI component they need. Build it as a proper Laravel Blade component.

1. **Read existing components** in `resources/views/components/` first to understand conventions
2. **Create the component class** in `app/View/Components/` if logic is needed, or use anonymous component (blade-only) for pure UI
3. **Create the Blade template** in `resources/views/components/`
4. **Style with Tailwind CSS** — match the project's design language (colors, rounded corners, shadows, etc.)
5. **Add Alpine.js** for interactivity if needed
6. **Show usage example** — how to use `<x-component-name />` in a page

## Common Components to Build

- Alert / notification banners
- Modal dialogs
- Stat cards (for dashboard KPIs)
- Data table with search + pagination
- Form input groups with labels and error messages
- Badge / status pills
- Sidebar nav items
- Dropdown menus
- Confirmation dialogs

## Steps

1. Understand what the component does and where it will be used
2. Read similar existing components or pages for design reference
3. Create the component file(s)
4. Provide a usage example
